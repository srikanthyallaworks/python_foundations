New Hotness: the dataclass

  r:int

  b:int

  def to_hex(self):
      return f'#{self.r:x}{self.g:x}{self.b:x}'

purple = Color(128,128,0)
print(purple.to_hex())
class Color:
  def __init__(self,red,blue,green):
      self.red=red
      self.blue=blue
      self.green=green

  def to_seq(self):
      return self.red,self.green,self.blue

  def to_hex(self):
      return '#' + bytes(self.to_seq()).hex()

Automated Tests
Definitions

Unit test: low-level test case written in the same language as the production code, which directly access its objects and members.

Surprising Controversy
  • Unit testing is surprisingly controversial
    • Probably because it’s hard to do well
    • Think of testing as a skill-- like SQL
  • Top complaints
    • Time to write
    • Time to maintain
    • Tests are worthless if they don’t get run

Reasons to Test
  1. Catch bugs
  2. Refactor with confidence
  3. Documentation that doesn't get stale
  4. Improved design

  • Tests are especially important in Python

    response = input(prompt)
    if response in ('y', 'ye', 'yes'):
        return True
    if response in ('n', 'no', 'nop', 'nope'):
        return False
    print(reminder)

Higher-Order Functions
  • A higher order function either
    • Takes a function as an argument
    • Returns a function as a result
  • Examples:
    • Built-in: map
    • Module: itertools
    • Module: functools
Passing Functions
def audited(to_audit):
  def wrapped(*args,**kwargs):
    print('starting')
    result = to_audit(*args,**kwargs)
    print('done!')
    return result
  return wrapped

def add(a,b):
  return a+b

_add = audited(add)

print(_add(11,22))

xs = [1,2,3,4]
squares = map(lambda x: x*x, xs)
Best Practice: Prefer Purity
  • A function is pure if:
    • It always returns the same thing given the same arguments
    • Doesn’t change state
  • Advantages:
    • Testable
    • Reusable
    • Easy to reason about
    • Doesn’t bork up shared state
