Python Philosophy
Vocab: Pythonic

The idioms of a programming language are defined by its users. Over the years, the Python community has come to use the adjective Pythonic to describe code that follows a particular style.

Pythonic Means...

Python programmers prefer to be explicit, to choose simple over complex, and to maximize readability.

Syntax Basics
Variables
  • Associate names with values
  • Pop into existence on assignment
  • Rules:
    • Only letters, numbers, and underscores
    • Don't start with a number
user_id = 12
label = 'Mailing Address'
xs = [0,118,2291]
address_1 = '1234 American Way'
Calling Functions
  • Invoke with parenthesis (always)
  • Pass arguments (sometimes)
  • Get something back (sometimes)
print()
print("Hello World!")

high_scores = [11,23,4,7]
result = sum(high_scores)
Defining Functions
  • Define with def
    def print_lyrics():
  print("I'm a lumberjack, and I'm okay.")
  print("I sleep all night and I work all day.")
  • Return values
    def add(a,b):
  return a + b
Indentation matters
  • Lots of languages use braces:
    function add(a,b){
  return a+b;
}
  • Python uses indentation:
    def add(a,b):
  return a+b
  • Easter egg: from __future__ import braces
  • https://docs.python.org/3/faq/design.html#why-does-python-use-indentation-for-grouping-of-statements
Empty Functions
  • The parser is confused by empty functions
    • Or classes or branches
  • Use pass as a placeholder
    def do_nothing():
pass
if true:
pass

Loop Constructs
  • Python loops come in just 2 flavors:
    • for
    • while
Using range
  • Lots of languages have a for loop:
    for(int i =0;i<10;i++){
  print(i);
}
  • Python uses range:
    for i in range(10):
  print(i)
  • https://realpython.com/python-range/
  • https://docs.python.org/3/tutorial/controlflow.html#the-range-function
For Variations
  • Kind of a foreach
    names=('Rob','Mary','David','Jenny','Chris','Imogen')
for name in names:
  print(name)

https://realpython.com/python-for-loop/

More like for

for i in range(10):
    print(i*i)
About range
  • Three arguments
    • Exclusive max
    • Inclusive min (defaults to 0)
    • Step (defaults to 1)
Fancy range
for i in range(1,10,2):
    print(i)
Keyword while
  • Executes as long as a condition is true
i = 0
while i < 3:
  print(i)
  i = i + 1
  • https://realpython.com/python-while-loop/
Kicking Out
  • return
  • break
i = 0
while True:
  print(i)
  i = i + 1
  if i>=3:
    break
print("done!")

Containers: Lists
List
  • Delimited with square brackets []
  • Mutable
  • Holds anything
words = ['here','are','words']
word_count = len(words)
print(f"There's {word_count} words")
Indexing
  • Access elements with brackets []
  • Zero based
  • Negative indexing: relative to the end
xs = ['it','is','like','a','weasel']
first_thing = xs[0]
second_thing = xs[1]
last_thing = xs[-1]
Adding Elements
  • Add single element with append
  • Add multiple elements with extend
    • Or the operator +=
nums = []
nums.append('three')
nums += ['four','five']
nums.extend(['six','seven'])
Removing Elements
  • Remove by index with pop
  • Remove by value with remove
bag = [1,2,3,'Chicken','Lizard']
bird = bag.pop(3)
bag.remove('Lizard')
Membership with in
names = ['bloggs', 'smith', 'patel']
if 'bloggs' in names:
  print('Joe is there.')
if 'kim' not in names:
  print('Adding patel'):
  names.append('patel')

Dictionaries
Dictionary

  • Delimited with curly brackets {}

When to use dictionaries
  • ID to Name mapping
  • Object to Count mapping
  • Name of a feature to value of the feature
Keys and Values
  • A value can be anything
  • A key must be hashable
    • Strings
    • Numbers
year_ratings = {
    2019: 'Ok',
    2020: 'Dumpster fire',
}
Data Access
  • Get & set values with square brackets []
calories = {}
calories['apple'] = 52
calories['mango'] = 89 
calories['butter'] = 546
print(calories['apple'] < calories['butter'])
